Act as a Senior Frontend AI Engineer. Your task is to build a highly advanced, gamified, and interactive Developer Portfolio. Do not use generic layouts; follow these exact technical specifications.

### 1. Architecture & Core Setup
- **Stack:** React 18, TypeScript, Vite, TailwindCSS v3.
- **Dependencies:** `framer-motion` (animations), `lucide-react` (icons), `@tsparticles/react` & `@tsparticles/slim` (background effects), `clsx` & `tailwind-merge` (utility styling).
- **Structure:** A single-page application (`App.tsx`) serving as the main layout, with gamification components placed in `src/components/`.

### 2. Design System: "Liquid Glass" (iOS 26 Aesthetic)
- **Theme Variables:** Use CSS variables in `index.css` for true Dark/Light mode toggling. Background should be deep space dark (`#0a0a0a`) or crisp white (`#ffffff`).
- **Liquid Glass Utility:** Create a `.liquid-glass` CSS class. It must have: `backdrop-filter: blur(24px) saturate(180%)`, a semi-transparent background (`rgba(255,255,255,0.05)` in dark, `rgba(255,255,255,0.6)` in light), and a 1px border of `rgba(255,255,255,0.1)`. 
- **Typography:** Modern sans-serif (e.g., Inter). Use subtle linear gradients on headings (e.g., `bg-gradient-to-r from-white to-gray-400 bg-clip-text text-transparent`).

### 3. Main Application (`App.tsx`)
Build a continuous scrolling layout containing:
- **Navigation:** A floating, pill-shaped `.liquid-glass` navbar at the top with a theme toggle (Sun/Moon).
- **Progress Bar:** A `framer-motion` `scaleX` progress bar fixed to the very top edge.
- **Hero Section:** Interactive `tsparticles` background. A dynamic greeting, a floating profile image, and prominent "Resume" and "Contact" action buttons.
- **Data Sections:** 
  - *Projects:* A grid of `.liquid-glass` cards with hover-lift effects.
  - *Experience & Education:* Minimalist timeline layouts.
  - *Skills:* Badge groups categorised by domain (AI/ML, Cloud, Web).
  - *Data Structure:* Store all text/links in a single `portfolioData` JSON object at the top of the file for easy updating.

### 4. Gamification 1: NLP AI Companion (`src/components/AICompanion.tsx`)
Place a floating orb in the **bottom-right** that expands into a `.liquid-glass` chat panel.
- **Input System:** A native text input field. NO external APIs. 
- **NLP Engine (Local):** Build a `generateResponse(query)` function. Convert the query to lowercase and use Regex word boundaries (`\bkeyword\b`) to match intents. 
- **Logic Matrix:**
  - Match `\b(contact|email|phone)\b` -> Return contact info.
  - Match `\b(projects|portfolio)\b` -> Return project list.
  - Match `\b(skills|tech)\b` -> Return tech stack.
  - Match specific keywords (e.g., `rag`, `aws`, `deepfake`) -> Return specific project details.
  - Fallback -> "Hmm, that isn't in my knowledge base."
- **UX:** Render AI text dynamically using a `setInterval` typing animation. Show a pulsing "AI is typing..." indicator.

### 5. Gamification 2: AI Core Reactor Game (`src/components/AICoreReactor.tsx`)
Place a glowing red orb in the **bottom-left**. Clicking it opens a full-screen `.liquid-glass` modal game.
- **Narrative Intro:** Display a flashing red `CRITICAL ALERT`. Tell the user: "The AI system crashed. As an AI Engineer, reconstruct the workflow to stabilize the core."
- **Split UI Canvas:**
  - *Left Panel:* A scattered pool of available tech modules (React, Python, Docker, Vector DB, etc.).
  - *Right Panel (Canvas):* A vertical, empty flowchart area.
- **Game Mechanics:** 
  - Clicking a module on the left moves it to the right canvas. 
  - Use `framer-motion` to dynamically draw glowing SVG arrows (`ArrowDown`) connecting the placed modules vertically.
- **Win/Loss States:**
  - *Target Sequence:* `Python → LangGraph → Gemini → Vector DB → AWS → Deployment`.
  - *Failure:* If a wrong module is clicked, trigger a red screen glitch animation (`x: [-10, 10, -10, 10, 0]`) and clear the canvas.
  - *Victory:* If the sequence is perfect, show a "Cooling Down..." state for 3 seconds, then transition the entire UI theme from Red to vibrant Green with the text "System Stable".

Ensure all components are strictly typed in TypeScript and exported to be rendered at the bottom of the `App` container.
